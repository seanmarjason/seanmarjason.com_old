Global Temperatures, what does this data look like?

23 June 2019

Something I’ve really wanted to explore and work with is global temperatures. I want to understand exactly what is going on in this crazy world, and become equipped with real information rather than constantly listening to the polarising views and opinions on global warming. Naturally I’m in the ‘we need to do something about this’ camp, but what exactly do we need to do? Well, hopefully data can help me to find some concrete actions I can take.

The data set that I am planning to work with is actually a consolidation of various sources of Global Temperature data put together by Berkeley Earth, which has been repackaged by Kaggle for use across the data community (Thanks Kaggle!). The data set can be found here:

https://www.kaggle.com/berkeleyearth/climate-change-earth-surface-temperature-data

Now, before I even get started working with this data, I guess I should at least understand what I’m working with? Time for some rough data exploration …

    So, what now? Well obviously this is a HUGE data set, so I need to be careful how I work with it.

    I’d like to start experimenting with some visualisations to see how I can best describe this data and potentially build something to help with data exploration without needing to write any code.

    I’d also eventually like to do some comparative analyses to understand the drivers of climate change.

    I’d also be keen to do something on climate change and global snowfalls, to see if I can understand what impact climate change is going to have on all the ski resorts that I’d like to visit in the coming years (please don’t take away that fresh POW!)

    Some key things that I need to be aware of through all of this:

  • There is inconsistent data in the earlier date ranges of this data set that I need to account for. I should look into this in more detail if I want to use the full data set to ensure that no analyses are skewed due to inconsistent data. I could achieve this with clever aggregations, or I may need to find more elegant solutions.
  • Given temperature readings are being take in individual locations across various countries, any aggregations will cause me to lose precision in my analyses. For example, in a single country like the US there will be a wide range of readings covering the snowy mountains of the north as well as the harsh southern climates, and so any averaging of temperatures in the US (and similar countries) may lose this large variation in temperatures.
  • Any comparisons of this data set to others need to account for the location that other measures or readings are taken. For example, the snowfall measurements that I would love to compare to are likely to be taken in different locations to these temperature readings, and so I need to ensure I am aware of this whenever comparing data sets like this.
